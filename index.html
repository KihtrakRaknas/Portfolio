File Retrieval


 In the process of developing an auton recorder, a need for a programming cable rose. Not only to debug the code by also so send recorded data back the computer. The following is a work around for the lack of a programming cable.

 An important niche detail about PROS is that when the cortex is unplugged from a controller or computer and then reinseted, the code runs from the beggining as if it was just power cycled but the files on the cortex stay. A key detail to remember is that the files are still stored in RAM which means they will disapear if you powercycle.

 Using this key detail, it is posible to write debuging information to a file during normal opperation and extract it afterwards.


Note:
 In order to reseive the file data after plugging the cortex into the computer you need to open the PROS terminal.

encodeMove


 
 When building a super accurate auton recorder, being able to act off of record sencor data is vital. This function allows a motor and a corresponding motor to get to any inputed state. If using this in an auton recorder, simply call this function until the function returns 0. At this point move on to replicating the next sensor value.

 Building an auton recorder in the way also allows for more compact recorded data. This method only need a final state for any action made, compared to the traditional recording every change to the motor's speed.


Note:
 V5 motors have encoders build into them so fucntions like this one will be useful and add zero extra cost or bulk to the robot
